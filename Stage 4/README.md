# Stage 4 — MVP Development

## 1. Overview

The goal of this stage was to begin implementing the Minimum Viable Product (MVP) based on the system design and technical documentation prepared in Stage 3.

Due to time constraints, the team focused on preparing the technical foundation and development structure rather than completing full implementation.

---

## 2. Development Approach

The team followed an Agile-based approach:

- Tasks were divided among team members based on roles
- Work was planned according to sprint goals
- GitHub was intended for version control using feature branches
- Testing strategy (QA) was defined alongside development

---

## 3. Work Completed

Although full development was not achieved, the following technical progress was made:

### System Design & Preparation
- Finalized system architecture (Frontend, Backend, Database, Notifications)
- Designed ERD (database schema)
- Created UML diagrams (Class & Sequence diagrams)
- Defined full data flow between system components

### API & Backend Planning
- Designed RESTful API endpoints:
  - Authentication (register/login)
  - Blood requests
  - Donations
- Structured backend logic using Node.js and Express (conceptually)

### UI/UX Design
- Designed main application screens:
  - Donor Home Screen
  - Patient Request Form
  - Hospital Dashboard
- Created interactive prototype

### Testing Preparation
- Defined QA strategy:
  - Unit Testing
  - Integration Testing
  - Manual Testing
- Simulated API requests using Postman (conceptual testing)

---

## 4. Work Not Completed

The following implementation tasks were not completed:

- Frontend development using React Native
- Backend API implementation and deployment
- Database setup and integration (MySQL)
- Firebase Cloud Messaging (FCM) integration
- Full system integration

---

## 5. Challenges Faced

- Limited time for full-stack implementation
- Lack of prior experience with some technologies
- Coordination between frontend and backend development
- Complexity of integrating multiple system components

---

## 6. Next Steps

If development continues, the following steps will be taken:

- Implement backend APIs using Node.js and Express
- Connect and test MySQL database
- Develop frontend screens using React Native
- Integrate Firebase notifications for real-time alerts
- Perform full system integration and testing

---

## 7. Summary

While the MVP was not fully implemented, the team successfully established a complete technical foundation, including system design, API structure, and user interface planning.

This foundation allows the project to be developed and completed in future stages.
